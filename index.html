JavaScript is a dynamic, untyped and interpreted programming lanaguage. It is prototype based with first class functions. Althrough it is commonly used in an object oriented style, JavaScript is a multi-paradigm language, supporting both imperative and functional styles.

Lexical Scoping


var value = "real";

function whatValue() {
	return value + " value";
}

whatValue();
> "real value"

Nested functions have access to the variables defined in the same scope or above.

Block Scoping (ES6)


if (true) {
	var hoistMe = "How rude!";
	let notMe = "Safe inside my block.";
}

console.log(hoistMe)
> "How rude!"

console.log(notMe)
> Reference Error

Non-function blocks don't get there own scope in JavaScript. This includes conditional blocks. But, ES6 intrduced the let statement, that specifically defines a variable as local to the block. Functions defined in blocks are allows block scoped starting in ES6. P.S. The reference error would only be thrown in strict mode. More on that later.

Closures


function outerFunc() {
	var innerVar = "hidden from global scope, unless...";

	function innerFunc() {
		return innerVar;
	}

	return innerFunc;
}

var func = outerFunc();
func();
> "hidden from the global scope, unless..."

Which leads to closures. A closure is created whenever a nested function references a variable in it's inherited scope. Essentially, the original scoping is maintained regardless of where the function is ultimately called.

New Function Syntax (ES6)


(function(...args) {
	console.log(args);
})(1, 2, 3);
> [1, 2, 3]

(function(x = 5, y = 10) {
	console.log(x + y);
})(null, undefined);
> 10

function destruct() {
	return [1, 2, 3];
}
var [a, b , c] = destruct();
console.log(c + ' ' + b ' ' + a)
> 3, 2, 1
The (function)() form is called an Immediately Invoked Function Expression IIFE. The spread '...' operator gathers all the arguments into an Array. This operator can also be used in non function definition contexts to break apart an array of elements. Default arguments. Null coerces to zero and undefined looks like it wasn't provided. Destructuring allows you to assign complex function returns to matching structures of variables. This also works for objects! Destructuring is a deep subject and there are a lot of ways to use it.

Custom Objects


function Person(name) {
	this.name = name;
}

var me = new Person("Dave");
me.name;
> "Dave"

Constructor functions executed with the new operator create new instances.

Let's Add Some Methods


Person.prototype.sayHello = function() {
	return "Hello, I'm " + this.name + ".";
}

Person.prototype.getHappy = function() {
	return this.name + " :)";
}

var me = new Person("Dave");
me.sayHello()
> "Hello, I'm Dave."

me.getHappy()
> "Dave :)"

Using special prototype property to add methods. 'this' within the function definition refers to the calling Object.

How About Inheritance?


function Student(name, subject) {
	Person.call(this, name);
	this.subject = subject;
}

Student.prototype = Object.create(Person.prototype);
Student.prototype.constructor = Student;

Student.prototype.sayHello = function() {
	return "Hello, I'm " + this.name + ". " +
		   "I'm studying " + this.subject + ".";
}

Functions have 3 special methods that allow 'this' to be set before the function is executed. call(), apply() and bind().

Object.create() creates a new object with the specified prototype. This allows us to create a prototype chain.

JavaScript is a prototypical language. All Objects have a prototype. When a member property is accessed, the local instance is checked and then each prototype in the prototype chain is checked. If no match is found, undefined is returned.
